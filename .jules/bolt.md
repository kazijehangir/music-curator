## 2024-05-14 - PocketBase N+1 Query Optimization
**Learning:** Pre-fetching records using PocketBase's `get_full_list` and iterating through files checking an in-memory dictionary lookup instead of querying `get_list` in an `os.walk` loop significantly reduces network latency and DB load, fixing an N+1 query issue during file discovery. Also, when pre-fetching by directory name (`source_dir`), remember to escape single quotes (e.g., `dir_name.replace("'", "\\'")`) so folders with quotes (like `80's Hits`) don't break the PocketBase filter string.
**Action:** Always map related records into memory using `get_full_list` (with field projections like `fields="id,file_path"`) to achieve O(1) lookups inside loops instead of O(N) queries, and proactively escape single quotes in filter string variables.
