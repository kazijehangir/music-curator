## 2024-05-24 - Command Argument Injection in Beets Import
**Vulnerability:** The `_pass_1_beets` function in `src/services/tagging.py` passed unsanitized user-controlled file paths to `subprocess.run` to execute the `beet import` command. If a file path began with a hyphen (e.g., `-v`), it could have been misconstrued as an argument/flag by the `beet` command, leading to command argument injection.
**Learning:** Even when the executable itself is safe, passing unvalidated or user-controlled strings as positional arguments to CLI commands without a separator can lead to unintended argument injection. This was present because the file paths, derived from user-managed directories, were implicitly trusted as non-options.
**Prevention:** Always use the `--` separator before positional arguments (like file paths) when passing user-controlled data to CLI commands via `subprocess`. This explicitly instructs the parser to treat all subsequent arguments as positional, neutralizing any leading hyphens.
